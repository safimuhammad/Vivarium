# Sprint 5.1 — Resident Memory Block (spec revision)

> Revises the Sprint 5 retrieval strategy. Supersedes "fold top-K memories into
> each perception turn" with a Claude-Code-style **always-resident memory block**,
> because top-K retrieval can silently miss important facts that don't rank high on
> a given breath. Builds on the shipped Sprint 5 foundation (files, scorer,
> reflection, identity) — this changes the *injection strategy*, not the store.

## 1. Why

Even salience-scored top-K surfaces a *subset* each breath, so an important fact
that doesn't rank top-K that moment is invisible. Keeping the whole memory resident
(up to a cap) guarantees nothing important is missed; retrieval is needed only for
the overflow beyond the cap.

## 2. Message layout (per breath)

```
[system]      identity (seed + self-narrative)              # unchanged
[user-synth]  MEMORY -- the resident block (<= cap)         # NEW, rebuilt at reflection
[assistant]   "These are the memories I carry."             # ack -> keeps the no-two-user invariant
... live perception -> decide -> tool turns ...             # appended; cache stays warm
```

- The block lives at a fixed position (lifecycle_history[1]=user, [2]=assistant ack),
  rebuilt **in place only at reflection** (and at init). Reflection already evicts the
  KV cache, so the rebuild is free; between reflections the block is byte-stable.
- `perceive()` reverts to a **pure sensory stream** (no memory fold-in) -- the block
  carries memory now.

## 3. Resident block contents (`MemoryStore.resident_block`)

- **memories <= cap:** the entire memory set (every memory resident; nothing missed).
- **memories > cap:** `[all HIGH-importance memories, always kept]` + fill the rest of
  the cap by full salience score (recency x importance x relevance to the
  reflection-moment perception) + a nudge line ("older memories exist; use `recall`").
  Reuses `select_memories(reserved = count of HIGH items, k = cap)`.
- Relevance in the over-cap fill is as-of the last reflection (refreshed each
  reflection, stale between -- acceptable: HIGH items are always resident regardless).

## 4. `recall(query)` -- overflow access tool

- An agent action-menu tool, but handled by the **Agent** (which owns `self.memory`),
  not the registry (registry tools have no memory handle) -- same pattern as the
  reflection tools. `Agent.execute` special-cases `recall` -> `MemoryStore.recall`.
- `MemoryStore.recall(query, current_breath, k)` = relevance-focused search over ALL
  memories (including overflow); returns top-k. Result rendered as a `tool` perception.
- Bonus path: the qwen3 spike (Probe A) showed the weak model under-uses tools, so the
  auto-curated block is the load-bearing path; `recall` is opportunistic.

## 5. Dashboard hook (deferred plumbing)

What the block/recall surface is emitted as a lightweight log signal now; the real
dashboard rendering ("RAG-surfaced associations alongside live turns") is a later layer.

## 6. Constants (new)

- `MEMORY_RESIDENT_CAP: int = 400` -- max memories resident in the block (~1 memory/line).
- `RECALL_K: int = 5` -- memories returned by a `recall` call.

## 7. Acceptance tests

1. Block under cap = all memories (ordered), no embedding needed.
2. Block over cap: size == cap; **every HIGH-importance memory is present** regardless
   of N (the guarantee); low-importance items compete for remaining slots.
3. Block rebuilt only at reflection; lifecycle_history[1]/[2] are the memory/ack pair;
   byte-stable across non-reflection breaths.
4. `perceive()` no longer injects memories (pure perception).
5. `recall(query)` returns relevance-ranked memories incl. overflow; empty/blank query
   handled; result appended as a `tool` turn with a paired id.
6. Agent offers `recall` in its action schemas; non-recall tools still dispatch to the
   registry unchanged.

## 8. Benchmark (the deliverable -- recall, retrieval, overflow)

Extend `bench/bench_memory.py`:
- **retrieval** -- `resident_block` build latency at under-cap and over-cap N.
- **recall** -- `recall(query)` latency vs N + quality (does it surface a planted
  memory by query?).
- **overflow** -- correctness: a HIGH-importance memory is always resident as N grows
  past the cap; block size stays == cap; a relevant low-importance overflow memory is
  reachable via recall though not resident.

Then optimize to targets and log before->after in the benchmark doc.

## 9. Scope guards (unchanged)

Action-chain compaction still Sprint 5.5; WorldState persistence still Sprint 6. This
revision bounds the *memory block* (cap), not the running `lifecycle_history`.
