# Sprint 5.5 — Transcript Compaction (design)

> Bounds the running `lifecycle_history` so an agent can breathe **forever** without
> ever overflowing the model's context window. Sprint 5.1 capped *memory*; this caps
> the *working transcript*. The single hard requirement: **the assembled prompt never
> exceeds the model's context window** (Safi's only acceptance bar).

## 1. Problem

Every breath appends ~3 turns (perception → decision+thinking → tool result) to
`lifecycle_history`, and qwen3's thinking can be large (~500–1000 tok/breath).
Nothing trims it. qwen3:8b's real window is **40,960 tokens** (Ollama clamps our
requested 65,536 to the model's trained max), and that window counts **prompt +
generation together**. Left unbounded, the transcript eventually crowds out room for
generation and the server silently truncates from the front — which would drop the
system prompt / memory block (catastrophic: the agent loses its identity). So the
transcript must stay bounded with headroom, for the life of the run.

## 2. Approach (decided)

A **Claude-Code-style running recap**: condense the oldest live turns into one
in-world "looking back" summary turn, keep the most recent turns verbatim, and drop
the rest. Fired on a **token-budget trigger that prefers reflection boundaries**
(reflection already re-prefills the KV cache, so compacting there is nearly free).
The recap is **cumulative** — each compaction summarizes (previous recap + newly
evicted turns) into a fresh recap, so it stays bounded.

Three cognition timescales, kept cleanly separate:

- **verbatim recent turns** — this moment (seconds);
- **running recap** ([3]/[4]) — the current episode (minutes);
- **memory + identity** (Sprint 5/5.1) — a lifetime — *unchanged by this work*.

## 3. History layout

```
[0] system     — identity                         (rebuilt at reflection)
[1] user       — resident memory block            (rebuilt at reflection)   [if memory]
[2] assistant  — memory ack                                                  [if memory]
[3] user       — "Looking back…" running recap     (rebuilt at compaction)   [if recap]
[4] assistant  — recap ack                                                   [if recap]
[5+]           — recent verbatim turns (start on a user/perception turn)
```

Both the memory pair and the recap pair are *optional* and tracked by flags
(`_resident_block_installed`, `_recap_installed`). The verbatim region begins at
`prefix_len = 1 + (2 if block) + (2 if recap)`. The recap pair is inserted directly
after the memory pair (or after the system turn for a memory-less agent). The kept
verbatim tail must begin on a `user` turn so the chat backend never sees two
consecutive `user` turns.

## 4. Budget math (the never-overflow guarantee)

Constants (in `core/constants.py`, the one tuned home):

| Constant | Value | Meaning |
|---|---|---|
| `MODEL_CONTEXT_TOKENS` | 40960 | qwen3:8b real window (the clamp ceiling) |
| `GENERATION_RESERVE_TOKENS` | 6144 | room reserved for the model's own output (thinking+reply) |
| `PROMPT_BUDGET_TOKENS` | = 40960 − 6144 = 34816 | max tokens the prompt may occupy |
| `COMPACTION_TRIGGER_TOKENS` | ≈ 0.70 × budget ≈ 24371 | compact when the estimate exceeds this |
| `COMPACTION_TARGET_TOKENS` | ≈ 0.50 × budget ≈ 17408 | compact *down to* roughly this |
| `COMPACTION_KEEP_RECENT_TURNS` | 8 | min verbatim turns always kept |
| `CHARS_PER_TOKEN` | 3.5 | estimation divisor — deliberately conservative (over-counts tokens) so the estimate errs toward compacting early; JSON tool schemas tokenize denser than prose |
| `COMPACTION_HARD_SAFETY_TOKENS` | ≈ 0.90 × budget ≈ 31334 | actual-prompt-token safety net |

**Guarantee.** After `_ensure_context_budget`, the estimated prompt ≤ target (17K)
≪ budget (34.8K) ≪ window (40.9K). The estimate counts **every text-bearing field**
(see `estimate_tokens` in §5) and uses a conservative divisor, so it tends to
*over*-count; even if it nonetheless under-counts, at a 2× miss the true prompt
approaches the budget ceiling — which is exactly the margin the actual-token net
below exists to catch (it is not "free" headroom). Two independent safety nets close
the gaps:

1. **Actual-token net.** Ollama returns `prompt_eval_count` (true prompt tokens).
   The loop captures it (`Decision.prompt_tokens`); if the *last* real prompt ever
   exceeded `COMPACTION_HARD_SAFETY_TOKENS`, the next breath force-compacts —
   self-correcting against estimator drift.
2. **Floor-overflow net.** Transcript eviction is *mechanical and unconditional*
   (see `compact()` in §5), so the verbatim region is always brought to ≤ target
   regardless of the LLM. If, after eviction to the `KEEP_RECENT` floor, the estimate
   is *still* over budget — only possible when `system + memory-block` alone are huge
   (a pathological, very large block) — the net degrades further: truncate the recap
   text, then truncate the in-context memory-block *turn* content (keeping
   HIGH-importance lines first + a "use recall" note). This never touches the durable
   store — only the rendered in-context copy — so the prompt is *always* made to fit.
   A CRITICAL log fires if `system + block` alone approach the budget (signal to
   lower `MEMORY_RESIDENT_CAP` for this model).

## 5. Components

- **`agents/compaction.py`** (pure data/strings, no I/O, no model):
  - `estimate_tokens(messages, tools) -> int` — **[C1]** for each message, sum the
    character length of *every text-bearing field*: `content` **and** `thinking`
    (assistant turns store hidden reasoning under `thinking`, not `content` — missing
    it under-counts by ~500–1000 tok/breath and breaks the guarantee). Add the
    serialized length of the `tools` schemas (they count toward the real prompt).
    Divide the total char count by `CHARS_PER_TOKEN` and round **up**.
  - `build_compaction_messages(prior_recap, evicted) -> list[dict]` — isolated
    `[system, user]` asking the agent to narrate, in-world, "what has recently
    happened to you," merging the prior recap with the evicted turns (DD9: no meta /
    "summarize"/"compact" language). Bounded by an explicit brevity instruction.
    **[L1]** When `prior_recap` is `None`/empty (first compaction), the prompt
    contains only the evicted turns — no "Earlier…" section, never the literal
    `"None"`.
  - `RECAP_ACK: str` — the assistant-voice ack placed at [4].
  - `truncate_to_tokens(text, max_tokens) -> str` — char-budget truncation helper
    for the floor-overflow net (keeps the head, appends an elision marker).

- **`agents/decider.py`**: `Decision` gains `prompt_tokens: int = 0`, populated from
  `getattr(response, "prompt_eval_count", 0) or 0` in `parse_ollama_response` (the
  count lives on the response, not `response.message`).

- **`agents/runtime.py`** additions:
  - New state in `__init__`: `_recap_installed: bool = False` and **[H2]**
    `_last_prompt_tokens: int = 0`.
  - Position helpers replacing the hardcoded index math:
    ```python
    def _prefix_len(self) -> int:
        return 1 + (2 if self._resident_block_installed else 0) + (2 if self._recap_installed else 0)
    ```
    `_set_resident_block` keeps inserting/replacing at [1]/[2] (the block is always
    the first pair after system; replacing it in place does not shift a recap already
    at [3]/[4]).
  - **[H1]** `_live_turns()` becomes `return self.lifecycle_history[self._prefix_len():]`
    so the reflection recap excludes BOTH the memory pair and the recap pair (else
    reflection would read the "Looking back…" narrative as a lived event — a feedback
    loop).
  - `_set_recap(text)` — insert/replace the recap pair `[user recap, assistant
    RECAP_ACK]` at index `1 + (2 if _resident_block_installed else 0)` (right after
    the memory pair, or after system for a memory-less agent); set `_recap_installed`.
  - `compact() -> None` — **[C2] eviction is mechanical and unconditional, done
    BEFORE any model call:**
    1. Compute the cut: evict the oldest verbatim turns until the estimate ≤ target,
       but always keep ≥ `KEEP_RECENT_TURNS`; snap the cut so the kept tail begins on
       a `user` turn **and** [chat-contract] never splits an assistant `tool_calls`
       turn from its paired `tool` results (evict whole breath-groups).
    2. Remove the evicted turns from `lifecycle_history` immediately (unconditional).
    3. Try: `recap = (await decider.decide(build_compaction_messages(prior_recap,
       evicted), tools=[])).text` — **[H3]** pass `tools=[]` so qwen3 returns
       narrative, not a tool call. On any exception, fall back to
       `recap = prior_recap or ""` (turns are already evicted, so we stay bounded).
    4. `_set_recap(recap)`.
  - `_ensure_context_budget(target) -> None` — if `estimate_tokens(...) > target`
    **or** `self._last_prompt_tokens > COMPACTION_HARD_SAFETY_TOKENS`, run
    `compact()`; then, while still over `PROMPT_BUDGET_TOKENS`, apply the
    floor-overflow net (truncate recap, then the in-context block turn) — guaranteed
    to terminate and fit.
  - Wiring in `breathe()`: after `perceive()` and before `decide()`, call
    `await self._ensure_context_budget(COMPACTION_TRIGGER_TOKENS)`. **[H2]** after a
    successful `decide()`, set `self._last_prompt_tokens = decision.prompt_tokens`.
    After `reflect()`, call `await self._ensure_context_budget(COMPACTION_TARGET_TOKENS)`
    (opportunistic — the cache is already cold, "prefer reflection").

## 6. Testing (TDD; deterministic, no live model)

- `estimate_tokens`: monotonic in content size; counts tools.
- Trigger: below trigger → no compaction; above → compaction fires (drive with a
  `MockDecider` and oversized turns; estimator is pure so it's deterministic).
- Recap pair lands at the right index for both memory and memory-less agents;
  verbatim tail starts on a `user` turn; no two consecutive `user` turns anywhere.
- Eviction: oldest verbatim turns removed, last `KEEP_RECENT` kept, recap installed
  with the prior recap folded in (cumulative).
- Idempotence: a second compaction replaces [3]/[4] in place (no duplicate pairs).
- **Never-overflow property test (the headline):** drive many breaths with
  worst-case-large turns through a `MockDecider`; assert the estimated prompt stays
  ≤ `PROMPT_BUDGET_TOKENS` on **every** breath.
- Floor-overflow net: with a pathologically large block + `KEEP_RECENT` floor,
  `_ensure_context_budget` still drives the estimate under budget (recap then block
  truncation).
- `Decision.prompt_tokens` parsed from `prompt_eval_count`; hard-safety net forces a
  compaction when the last real prompt was too big.
- Reflection still works with the recap pair present (`_live_turns` must also exclude
  the recap scaffolding from the reflection recap).

## 7. Benchmark (the deliverable)

Extend `bench/bench_memory.py` (or a sibling) with a **never-overflow** section:
drive a synthetic long run (hundreds of breaths) with realistic + worst-case turn
sizes and report: peak estimated prompt tokens, compaction count, mean turns evicted,
and — on the real chroma/qwen3 path — the peak **actual** `prompt_eval_count` vs the
window. Headline pass condition: **peak actual prompt + generation never reaches
40,960.** Tune the ratios to maximize retained verbatim context subject to that.

## 8. Scope guards

No change to the memory store, retrieval, reflection write-path, or world. Compaction
is purely an in-context transcript transform on `lifecycle_history`. Multi-agent
runner / persistence remain Sprint 6.

## 9. As-built deltas (post-implementation + code review)

The shipped implementation matches the spec; these refinements emerged during the
build and the code-review pass and are recorded here so the spec stays canonical.

- **Generation cap closes the OTHER half of the window math.** The window counts
  prompt + generation together, so bounding the prompt alone (§4) is insufficient — a
  long generation could still overflow. `OllamaDecider` now sets
  `num_predict = GENERATION_RESERVE_TOKENS` (6,144). With
  `PROMPT_BUDGET = window − reserve`, `prompt + generation ≤ window` by construction.
- **The floor net is now provably terminal.** `_enforce_prompt_budget` degrades in
  order — shrink recap → shrink memory block → drop oldest breath-groups → **shrink
  surviving turns newest-first (system turn last)**. The final stage was hardened after
  review: a lone surviving turn (a huge perception is NOT bounded by the generation
  reserve) or oversized scaffolding is shrunk until the prompt fits. The only
  irreducible floor is the tool schemas themselves; if they alone exceeded the budget
  the agent logs `CRITICAL` rather than returning silently over budget (cannot occur
  with the current handful of small schemas).
- **compact() is best-effort, enforcement is mandatory.** `_ensure_context_budget`
  wraps `compact()` in try/except + `logger.exception`, so a failed/slow recap model
  never skips the mechanical floor net. Eviction is deterministic; recap text is
  best-effort prose layered on top.
- **compact() never evicts the most-recent perception.** Guard widened to
  `cut <= 0 or cut >= len(verbatim)` so the agent never acts blind.
- **Estimator conservativeness — settled empirically.** On real qwen3:8b the estimate
  is ≥ the model-reported `prompt_eval_count` on every breath, by ~1.4–3.5× (the gap
  widens with transcript size). The estimator over-counts via both the 3.5 chars/token
  divisor and the serialized JSON structure the chat template does not emit as tokens.
  A forced-compaction live run (`--force-compaction`) confirms compaction firing on the
  real model authors recaps, keeps the prompt under budget every breath, and the agent
  stays ALIVE. Evidence: `docs/superpowers/benchmarks/2026-06-28-compaction-perf.md`.

Analyzed and ruled non-bugs (no change): the resident block is installed at init but
`_set_resident_block` no-ops for NULL-memory agents; `RECAP_ACK` is not shrunk by the
floor net (bounded and minor).
